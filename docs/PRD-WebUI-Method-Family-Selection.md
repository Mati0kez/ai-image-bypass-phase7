# PRD: WebUI 方法族自定义多选与智能联动功能

## Problem Statement

用户在进行方法族对抗性逐项测试时，需要能够**精确控制启用哪些方法族**（例如只测试 `frequency` 或 `noise + lpips` 的组合），同时希望界面能智能地提示哪些参数与当前选中的方法族相关，避免无效参数设置导致的困惑。

当前 WebUI 虽然支持 `profile` 选择，但缺乏细粒度的**方法族多选**能力，且参数滑块与方法族之间没有联动关系，用户体验较差。

## Solution

在 WebUI 的「高级对抗选项」中实现以下能力：

- 将 10 个方法族按功能分组展示（基础方法族 / 对抗方法族 / 重生成方法族）
- 支持用户任意多选方法族
- 根据用户选中的方法族，**动态启用/锁定**相关的参数滑块
- 增加「画质优先模式」，勾选后自动将强度参数调整为推荐低值并锁定滑块

## User Stories

1. 作为测试者，我希望能只勾选 `frequency` 方法族，界面自动锁定其他不相关的强度滑块。
2. 作为研究者，我希望能同时开启 `noise + lpips_blackbox`，并看到相关参数保持可编辑。
3. 作为用户，我希望开启「画质优先模式」后，所有强度参数自动降低并禁用编辑。
4. 作为评估者，我希望方法族选项使用中英文对照，方便理解。

## Implementation Decisions

- 在 `webui_bypass.py` 中新增三个 `CheckboxGroup`（基础 / 对抗 / 重生成）
- 定义 `METHOD_PARAM_MAP`，建立方法族与参数的映射关系
- 使用 `gr.on` / `.change` 事件实现前端联动逻辑
- `TransformConfig` 新增 `methods: Optional[list[str]]` 字段
- `pipeline.py` 支持优先使用用户自定义的方法族列表
- 画质优先模式采用前端联动方式（勾选后立即修改滑块值并锁定）

## Testing Decisions

- 手动测试单方法族勾选场景，验证滑块锁定行为
- 手动测试多方法族组合场景
- 手动测试画质优先模式开关对滑块的影响
- 验证 manifest 中 `method_families` 字段是否正确反映用户选择

## Out of Scope

- 自动化消融测试报告生成
- CLI `--methods` 参数增强
- 与真实外部检测平台（Hive）的集成测试

## Further Notes

本 PRD 记录的是 Phase 7 后期对 WebUI 的增强，目的是支持用户进行**方法族级逐项手动测试**。后续可在此基础上扩展自动化测试框架。
